Features
Everything you need, nothing you don't
🔒

Complete Auth Flow

Login, registration, lost password, and password reset — all handled on your frontend with proper nonce verification and WordPress hook compatibility.

⚡

Elementor Widgets

Four native Widget_Base widgets with full Elementor style controls, V4 compatibility, dynamic content flagging, and Theme Builder integration via real WordPress pages.

🛡

Rate Limiting

Configurable per-IP lockout after failed attempts. Uses transients with anonymised IP hashing. Protects login, registration, lost-password, and reset-password forms.

🕷

Honeypot Protection

Rotating hidden field (hourly key rotation via HMAC) catches bots. Trapped submissions get a fake success response — bots never know they failed.

🔌

AJAX Forms

Optional no-reload form submission. Inline error display, loading spinners, and automatic redirect on success — all configurable from the admin panel.

🌐

URL Rewriting

All wp-login.php links across your entire site — login_url, logout_url, lostpassword_url, site_url — are transparently rewritten to your frontend pages.

👥

Multisite Ready

Network activation support. Per-site settings, signup/activation URL rewriting, and automatic option seeding for new sub-sites via wp_initialize_site.

♻

3rd-Party Compatible

Fires standard WordPress hooks (login_form, register_form, etc.) so 2FA, CAPTCHA, and social login plugins render their fields inside your forms automatically.

♿

Accessible

ARIA roles, aria-required, aria-live regions for errors, focus-visible outlines, and translatable password-strength labels. Clean, semantic HTML throughout.

Quick Start
Up and running in 60 seconds
# Upload and activate
wp plugin install wp-frontend-auth.zip --activate

# Flush rewrite rules
wp rewrite flush

# Link existing Elementor pages (or use "Create Pages" in Settings)
# If you built your auth pages manually, this detects and links them
# without creating duplicates. Then flush rules again.

# Optional: enable AJAX forms and user-chosen passwords
wp option update wpfa_use_ajax 1
wp option update wpfa_user_passwords 1

# That's it. Visit /login/ on your site.
Changelog
What's new
v1.4.16 Latest
  • Critical Fixed rewrite-rule flush timing — was scheduled on shutdown, so the flush only took effect on the next request. /login/ would 404 on the first load after an upgrade or FTP deploy. Flush now runs on init priority 99.
  • Critical Fixed stale page-ID option causing hard 404s. wpfa_get_page_id() now verifies the stored ID refers to an existing published page, clearing the option if not — so the rewrite/virtual-page fallback can take over instead of silently failing.
  • High Fixed blank page on Elementor-built auth pages with no widgets. Elementor outputs empty wrapper divs that prevented wpfa_maybe_inject_form() from injecting the form. Now uses wp_strip_all_tags() to test for visible text rather than raw HTML length.
  • High Fixed MCP Bridge / OAuth redirect loop. wpfa_filter_site_url() was not applying the same REST exemption as wpfa_filter_login_url(). The MCP Bridge calls site_url('wp-login.php') directly, bypassing the existing exemption and pointing OAuth redirects at the WPFA frontend page.
  • Medium Added cache exclusion for auth pages at template_redirect priority 0 via Cache-Control: no-store, X-LiteSpeed-Cache-Control, DONOTCACHEPAGE, DONOTROCKETOPTIMIZE, and related constants for WP Rocket, W3 Total Cache, and WP Super Cache.
  • Medium Added cache purge on version upgrade. On update, auth page URLs are purged from LiteSpeed Cache, Super Page Cache, WP Super Cache, and WP Rocket — clearing any stale 404 responses cached before rewrite rules existed.
v1.4.15
  • Critical Added wpfa_maybe_inject_form() — auto-renders the auth form on virtual pages and empty real WPFA pages. Previously the theme rendered a blank page unless a widget was placed manually.
  • Medium Fixed Undefined array key "key" / "login" PHP warnings in the Reset Password widget when URL parameters are absent.
v1.4.14
  • High Security: fixed PHP 8.0+ fatal TypeError DoS via array-valued HTTP parameters across 7 files.
  • Medium Security: added missing honeypot check to lost-password handler — was renderable but never checked, allowing bots to trigger mass password-reset emails.
